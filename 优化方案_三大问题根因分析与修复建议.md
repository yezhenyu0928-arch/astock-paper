# astock-paper 三大问题根因分析与修复建议

> 分析日期：2026-07-16　范围：看板无法访问 / 策略无交易 / 微信推送异常
> 说明：本文档只做**根因分析 + 修复建议**，未改动任何代码。确认方案后再实施。

---

## 一、总览：三大问题其实同源

三个现象看似独立，实际有一条共同的根因链：

```
数据层缺口(stock_annual空/本地断更) + 消息面L0词表误判(全球快讯"制裁/加息")
        │                                          │
        ▼                                          ▼
  依赖ROE/基本面的策略全军覆没            市场分被误打到 -2（min(L0,L1) 只降不升）
        │                                          │
        ▼                                          ▼
  候选池为空 → 无买单           exposure_mult=0 抹平买单权重 + 推送"冻结开仓"
        └──────────────┬───────────────────────────┘
                       ▼
          账户长期空仓、nav 恒=1.0、推送内容与A股无关
```

Chart.js 走 `cdn.jsdelivr.net` 是**看板"看起来打不开"**的独立诱因。

## 二、问题1：看板无法访问

### 2.1 实际状态
线上 `https://yezhenyu0928-arch.github.io/astock-paper/index.html` **实际可加载**，数据已更新到 2026-07-16。所以"完全打不开"更可能是**图表区域空白**造成的"看起来打不开"，或 Pages 偶发停更。

### 2.2 根因
| # | 根因 | 位置 | 影响 |
|---|------|------|------|
| A | 收益曲线依赖境外 CDN `cdn.jsdelivr.net` 加载 Chart.js | `report_html.py:1047` | 中国大陆该 CDN 常被墙 → 图表空白、页面"半死"，用户误判为打不开 |
| B | 无独立 Pages 部署 workflow，仅靠 `daily.yml` 把 `docs/` 提交到 main 由仓库 Pages 服务托管 | `.github/workflows/daily.yml` | daily 任一步失败 / Pages 分支路径配置错 → 看板停更且无独立告警 |
| C | 海外 Runner 数据源不可达时主流程"跳过质检继续"，看板可能拿旧数据渲染 | `run_daily.py:192-193` | 看板显示滞后数据，易被当成"挂了" |

> 注：`report_html.py:1495` 的实时价 `_LIVE_JS` 已用国内 `qt.gtimg.cn` 且有 3s 超时兜底，这部分没问题；问题只在 Chart.js。

### 2.3 修复建议
1. **【P0】Chart.js 本地化**：把 `chart.umd.min.js`（约 200KB）下载到 `docs/vendor/chart.min.js`，`report_html.py:1047` 改为引用本地相对路径 `./vendor/chart.min.js`。彻底摆脱境外 CDN，且 GitHub Pages 本身国内可达。
   - 备选：换国内镜像（如 `https://cdn.bootcdn.net/ajax/libs/Chart.js/4.4.7/chart.umd.min.js`），但仍有第三方可用性风险，不如本地化稳。
2. **【P1】图表降级兜底**：在 `<canvas>` 外包一层 `<noscript>`/JS 探测，若 `window.Chart` 未定义则渲染一张纯 HTML 表格版收益率，保证"再差也有数字看"。
3. **【P1】独立 Pages 部署 + 停更告警**：新增 `pages.yml`（`actions/deploy-pages`）或在 daily 末尾校验 `docs/index.html` 的 mtime；已有 `run_daily.py:276` 的"看板生成失败告警"，可补一条"看板超 N 天未更新"的心跳自检。
4. **【P2】质检降级可视化**：`run_daily.py:192` 跳过质检时，在看板顶部打一个"⚠ 数据源不可达，展示为缓存快照(截至X)"的醒目 banner。

## 三、问题2：策略无任何交易

### 3.1 实际状态
`trade_log` 仅 1 条（2026-07-03 信号、07-06 成交，s2_etf 买中证500ETF 5400股），之后所有账户 `nav≡1.0`、`pending` 为空。

### 3.2 根因（按贡献度排序）
| # | 根因 | 位置 | 说明 |
|---|------|------|------|
| A | **消息面市场分被误打到负值 → 敞口系数=0，抹平所有买单** | `news_engine.py:147` `market_exposure_mult`；`risk.py:103-126` post_check 规则6 | `exposure_map[-2]=0.0 / [-1]=0.5`。市场分=-2 时 `o.weight*0→0→continue`，买单被静默删除。这是"全员空仓"的最大黑手（详见问题3根因）。 |
| B | **price-based 大盘冻结删所有 buy** | `risk.py:54-66` pre_check + `risk.py:112-115` post_check 规则1 | 沪深300 单日跌>3% 或 20日跌>10% → `market_frozen=True` → 删所有买单。线上"沪深300近1月-3.2%、中证500-7.8%"，20日跌幅接近阈值时会周期性触发。 |
| C | **stock_annual 空/覆盖不足 → 依赖ROE的策略候选池为空** | `s1_dividend.py`（v2 需连续3年 ROE>8%）；本地库 stock_annual 仅覆盖 43/300 只 | ROE 质量过滤把绝大多数候选剔除 → S1 几乎永远选不出票。`.backtest-trigger` 已记录此为历史根因，backtest.yml 的 gate 已修，但 daily 主流程 `run_daily.py:162` 的 `update_annual_roe` 仅在 `month<=5` 执行，7月后不补年报数据。 |
| D | **调仓周期错配**：多数策略非每日交易 | `s2_etf.py:19` 每周最后交易日；`s1_dividend`/`s7_track` 月末；`s6_sector` 月度 | 平日本就不产生信号，叠加 A/B/C 后"看起来永远不动"。S2 已持有中证500ETF 且非换出条件，故无新单属**正常**。 |
| E | 本地 DB 是旧快照（daily_bar 止于 07-10），非线上真实状态 | 本地 `db/market.sqlite` | 排查时注意区分"本地快照"与"线上 CI 实际结果"，避免误判。 |

### 3.3 修复建议
1. **【P0】敞口系数不应"一刀切归零"**：把 `exposure_map` 的 `-2` 从 `0.0` 改为 `0.3`（重度减仓但不封死），`-1` 保持 `0.5`。同时在 `risk.py` post_check 规则6 增加日志，记录"因消息面被削/删的单"，让"为什么没交易"可追溯。
2. **【P0】修复市场分误判的上游**（见问题3，这是治本）。上游修好后 B、A 会大幅缓解。
3. **【P1】大盘冻结阈值与语义修正**：
   - `m20_drop=0.10`（20日跌10%才冻结）本身合理，但 `day_drop=0.03` 偏敏感（A股单日-3%不算罕见）。建议 day_drop 放宽到 `0.05`，或改为"单日-3% 且 跌破MA20"双条件，减少误冻结。
   - 冻结时应**只禁新开、保留调仓/减仓**，当前 post_check 规则1 删所有 buy 会连"轮动换入"一起杀掉，导致 S2 想换更强ETF也换不了。建议区分"新开仓"与"轮动置换"。
4. **【P1】年报数据补齐常态化**：`run_daily.py:162` 的 `update_annual_roe` 去掉 `month<=5` 限制，或改为"检测 stock_annual 覆盖率<80% 就补"，与 backtest.yml 的 gate 逻辑对齐。
5. **【P2】空仓可观测性**：心跳推送里补一句"今日无操作原因分类"（冻结/无信号/候选空/调仓日未到），避免用户长期看不到交易而怀疑系统故障。

## 四、问题3：微信推送"冻结开仓" + 理由与A股无关

### 4.1 根因（核心，也是 problem2 的上游）
**市场分被系统性打负 → 触发误导性推送**，两个叠加机制：

| # | 根因 | 位置 | 说明 |
|---|------|------|------|
| A | **L0 词表命中"全球快讯"里的国际新闻，误判为A股重大利空** | `news_engine.py:27-31` 词表；`news_adapter.py` fetch_flash 抓取东财全球快讯 | `MARKET_NEG` 含 `制裁(-2)`、`开战(-2)`、`加息(-1)`、`地缘冲突升级(-2)`。全球快讯里美联储"加息"、国际"制裁/冲突"几乎天天出现 → L0 轻松打到 ≤-2，**但这些与A股走势无关**。这就是"理由内容与A股逻辑不符"的直接来源。 |
| B | **L0 与 L1(GLM) 取 min，只降不升** | `news_engine.py:72` `elif l1s < score: score = l1s` | 最终市场分 = min(L0, L1)。任一层偏负，结果就偏负；GLM 即使判中性也拉不回来 → **系统性负偏**。 |
| C | **盘中扫描推送文案写死"今日主流程将冻结开仓"** | `run_intraday.py:129-131` | `score<=-2` 即推 `🔴盘中市场级重大负面(分X)：[国际headline]，今日主流程将冻结开仓`。而主流程真正的冻结是 **price-based**（risk.py），与这条news分**并不等价** → 文案误导。 |
| D | pre_check 的"🔴大盘冻结…禁止开仓"其实**没被推送** | `engine.py:544` 拿到 `pre["alerts"]` 后**丢弃**，`run_daily.py` 未 push | 说明用户看到的"冻结开仓"字样几乎全来自 C（盘中扫描），而非日终主流程。 |

### 4.2 修复建议
1. **【P0】市场级新闻只用"A股相关"语料**：`scan_market` 读 `news_raw` 时，来源过滤掉"全球/国际"快讯，或对国际词条降权。具体：
   - 给 `MARKET_NEG` 的国际类词（制裁/开战/加息/地缘冲突）加"是否含A股主体"的判定，或直接把它们从**市场级**词表移到"仅参考、不计分"档。
   - `news_adapter.fetch_flash` 对全球快讯打 `scope=global` 标签，`scan_market` 默认只统计 `scope=domestic`。
2. **【P0】L1 不再只做 min，改双向校准**：`news_engine.py:72` 逻辑改为"L0 与 L1 分歧过大(如 |L0-L1|≥2)时，取二者均值或以 L1 为准并降权"，避免任一层噪声单边压制。GLM prompt(`prompts/news_daily.txt`)补充"仅对影响A股的国内政策/资金/监管事件打分，国际事件除非直接冲击A股否则记0"。
3. **【P0】盘中推送文案与事实对齐**：`run_intraday.py:130` 文案改为"🟡盘中检测到潜在利空信号(分X)，仅供参考；是否冻结以日终主流程 price-based 风控为准"。不要断言"主流程将冻结"。
4. **【P1】推送理由带可解释证据**：市场分为负时，推送里附"命中词/原始标题/来源"，用户一眼能看出是不是国际新闻误判（当前 `ev` 已有，但需在文案里透出并标注来源域）。
5. **【P1】增加"冻结/减仓"去抖**：连续 N 次扫描都≤-2 才推一次，避免同一利空反复轰炸。

## 七、方案完成后的实际变化：是否付费 / 是否必须开机 / 是否零维护

> 以下结论均基于对 `config.yaml`、`data.py`、`data_adapter.py`、`news_llm.py`、`notify.py`、`.github/workflows/*.yml` 的实际核查，不是猜测。

### 7.1 要不要付费？——可以做到 **¥0/月**

| 环节 | 当前实现 | 费用 | 说明 |
|------|----------|------|------|
| 行情/基本面数据 | `akshare`（主）+ 腾讯 `stock_zh_a_hist_tx`（个股主源）+ `baostock`（兜底），`tushare` 仅作"有积分才用的增强源" | **免费** | `data_adapter.py:43` 注释明确"个股主源改腾讯(免费无 token)，Tushare 次之(需积分,无则自动跳过)"；无 token 时优雅跳过，不影响运行 |
| 新闻打分 LLM | 智谱 `GLM-4.7-Flash`，注释标注"免费" | **免费** | 关键：已验证 `news_engine._l1_market` 把 `news_llm.market_score` 包在 `try/except` 里（`news_engine.py:115-120`），失败返回 `None` 并回退 L0 规则。**即 GLM 免费额度变动/密钥失效也不会让每日任务崩溃**，只会退化为"纯规则打分" |
| 微信/邮件推送 | PushPlus（主）+ QQ 邮箱 SMTP（备） | **免费** | PushPlus 免费额度对"每天几条"足够；SMTP 走 QQ 邮箱永远免费，作兜底 |
| 计算/调度 | GitHub Actions（daily 17:40/18:10、intraday 10:00/13:30/14:45、keepalive 保活） | **免费** | 公开仓库无限免费；私有仓库每月 2000 分钟免费，本系统约 10–15 分/天 ≈ 450 分/月，远低于上限 |
| 看板托管 | GitHub Pages | **免费** | 静态托管，国内可访问 |
| 数据缓存 | GitHub Actions cache（10GB，keepalive 防 7 天驱逐） | **免费** | — |

**结论：全链路无付费订阅即可跑。** 没有任何"不交钱就停"的硬依赖。

### 7.2 必须电脑开机吗？——**不需要**

整条流水线（拉数据 → 质检 → 跑引擎 → 推送 → 提交看板）**全部在 GitHub 服务器上按 cron 自动跑**。你的电脑只在"改代码 / 本地调试"时需要开机。
关机、出差、断网，都不会影响每天自动运行与推送。本地 `db/market.sqlite` 只是开发快照，线上结果以 CI 为准。

### 7.3 能做到"不需要维护"吗？——**接近零维护，但需做到"故障可感知"**

诚实说：任何依赖外部免费 API 的云程序，都无法承诺"绝对零维护"（akshare 接口偶尔漂移、GLM 免费额度可能调整、PushPlus 账号可能异常）。但可以把维护成本压到极低，并让故障**吵醒你**而不是静默无交易：

- **已具备的稳健性**：LLM 失败自动回退 L0（不会崩）；多数据源互相兜底（新浪/腾讯/baostock）。
- **还需补的 3 个加固（纳入 P0/P1）**：
  1. **运行失败告警**：当前只有"DB 丢失"告警（`run_daily.py:139`），缺"工作流运行失败"通知。`daily.yml` 增加 `if: failure()` 步骤，用 PushPlus/邮件告诉你"今天没跑成"，而不是你几天后看空仓才发现。
  2. **推送主通道去依赖**：PushPlus 账号若被限，自动切到永远免费的 QQ 邮箱 SMTP（代码已有 SMTP 备用逻辑，确保它真能兜住）。
  3. **关键 bug 一次性修**：`update_annual_roe` 的 `month<=5` 限制（`run_daily.py:162`）— 这是分红/质量策略长期选不出票的根因，修一次即永久消除。
- **一次性动作**：首次部署需先跑一次 `backfill` 工作流建全量历史库（README 步骤 6），此后 cache 保活即可。

**结论：免费 ¥0 + 不依赖本机开机 + 接近零维护 + 故障可感知**，是本方案的目标状态。剩下的一点点"偶尔修一下外部 API 漂移"无法归零，但已不构成负担。

## 八、市场新闻识别与打分标准 v2（合理的量化指标设计）

> 用户诉求：消息来源"国家领导人/证监会"应更权威；参考 GitHub / 微信公众号上业界怎么给新闻打分；做成**合理、可追溯的量化指标**。
> 设计参考了业已存在的成熟做法：
> - GitHub `newsdim`（8 维打分 mom/stab/horz/eng/hype/sent/sec/pol，各 -3..+3）；
> - GitHub `A_Share_investment_Agent`（情感分 -1..+1 + 置信度，含"重大利好/行业政策"分级）；
> - 券商/舆情实践 `新联财通四维`（媒体影响力 A+央党媒85分 vs C自媒体20分）、`信源可信度矩阵`（最终情感分 = 原始分 × 信源可信度，拦截 87% 自媒体误导）、`经济体温计`（SOURCE_WEIGHTS：央行类 1.5 / 顶级财经 1.2 / 行业垂直 0.8）；
> - 国泰君安期货研究：多源融合 + LLM 语义去噪 + 动态因子，与沪深300 有 1–2 日领先性。
>
> 核心改进：**从"泛化词表命中即计分"升级为"信源分级 × 事件方向 × 衰减"的三维量化模型**，直接消除"国际新闻误冻全市场"与"理由与 A 股无关"两个痛点。

### 8.1 维度一：信源分级权重（谁说的更算数）

| 级别 | 信源范围 | 权重区间 | 说明 |
|------|----------|----------|------|
| **S0 最高权威** | 国务院/政治局会议/中央经济工作会议通稿、央行(PBOC)货币政策、证监会(CSRC)官方发布、新华社/人民日报/新闻联播通稿、财政部、发改委 | **1.5** | 用户最关心的"国家领导人/证监会"落在此级，单条即可主导市场分方向 |
| **S1 官方/交易所** | 上交所/深交所/北交所公告、外汇局、国资委、国家统计局 | **1.2** | 监管文件、退市/减持新规等 |
| **S2 主流财经媒体** | 央视财经、经济日报、财新、证券时报、券商中国、21世纪经济报道 | **1.0** | 基线权重 |
| **S3 市场媒体/快讯** | 东方财富、新浪财经、同花顺、财联社 | **0.7** | 快讯噪声大，仅作微调 |
| **S4 自媒体/社交** | 微信公众号、雪球、股吧、微博 | **0.4** | 仅参考，权重最低，且需"正文三元组校验"防标题党 |
| **国际信源**（路透/彭博/东财全球快讯） | — | **×0.3 折扣，且 scope=global 单列** | **这是修复原 bug 的关键**：国际事件只对"直接涉华/涉 A 股"才计分，且整体打 0.3 折，避免"美联储加息/海外冲突"把 A 股市场分拖到 -2 |

> 实现：在 `news_adapter.fetch_flash` 给每条新闻打 `source_tier` 与 `scope`（domestic/global）标签；`scan_market` 读取后按上表映射权重。

### 8.2 维度二：事件方向定性（A 股专属事件词典，不再是泛化"制裁/开战"）

用确定性规则词典给每条新闻一个**基础分 base_i ∈ [-2, +2]**，按 A 股实际影响方向定，而非英文舆情那套"战争=-2"：

| 事件类别 | 触发词/情形 | 基础分 |
|----------|-------------|--------|
| 货币宽松 | 降准 / 降息 | **+2** |
| 货币宽松（预期） | 降准预期 / 窗口指导 / 流动性呵护 | **+1** |
| 货币收紧 | 加息 / 提准 / 回收流动性 | **-2** |
| 资本改革·利好 | 印花税下调 / 汇金等"国家队"增持 / IPO 放缓 / 再融资收紧 / 减持新规从严 | **+1 ~ +2** |
| 资本改革·利空 | 印花税上调 / IPO 提速 / 再融资放开 / 大股东减持松动 | **-1 ~ -2** |
| 财政/产业·利好 | 特别国债 / 财政发力 / 产业扶持基金（半导体/新能源/AI/消费） | **+1 ~ +2** |
| 财政/产业·利空 | 加征关税 / 出口管制 / 产业打压 | **-1 ~ -2** |
| 监管执法 | 退市常态化 / 严查违规 | **-1（短期情绪，长期中性）** |
| 外部冲击（非涉华） | 美联储激进加息 / 地缘冲突（未直接涉 A 股） | **-1 ~ -2 × 国际折扣 0.3** |
| 外部冲击（涉华制裁） | 直接打击特定板块的制裁 | **-2 但精确打击板块，不波及全市场分** |
| 个股黑天鹅 | 财务造假 / 被立案 / 退市风险 | **-2（个股级，不计入市场分）** |

> 与原 L0 的本质区别：原词表把"制裁/开战/加息"一律 -2 且不分是否涉 A 股；新词典只把"直接冲击 A 股的国内政策/资金/监管"给高分，国际事件单列折扣。

### 8.3 维度三：聚合与输出（不再"只降不升取 min"）

对当日 N 条新闻，每条得到 `贡献分_i = base_i × w_i`（w_i 截断到 [0.3, 1.5]）：

```
market_score = clip( tanh( Σ_i w_i · base_i / N_eff ) × 2 , -2, +2 )
```

关键修正（对应原 bug B）：
- **权威源主导、低权源封顶**：S0/S1 的 `base_i` 直接进分子；S3/S4 的 `base_i` 对最终分的贡献**封顶 ±0.5**（避免股吧/自媒体噪声单边拖动）；国际源（×0.3）只在"直接涉华"时计入。
- **GLM 降级为"校准器"而非"决策者"**：只对 S2/S3 中规则模糊的事件用 LLM 给 `base_i`；S0/S1 权威事件**用规则确定性给分**，LLM 仅做一致性校验。彻底去掉 `min(L0, L1)` 的"只降不升"，改为"权威规则定方向、LLM 微调模糊项"。
- **衰减**：事件分在 1–3 个交易日按 `exp(-t/τ)` 衰减（文本因子半衰期约 1–3 天，业界共识），避免旧消息长期压制。
- **可解释输出**：每条新闻落 `(来源分级, 事件类型, 基础分, 权重, 贡献分)`。推送市场分为负时，文案透出"权威信源X：事件Y（分Z）"，让用户一眼看出是不是国际新闻误判 —— 直接回应"理由与 A 股逻辑不符"。

### 8.4 直接对应你之前的两个痛点

| 痛点 | v2 如何消除 |
|------|-------------|
| "推送理由与 A 股逻辑不符" | 打分基于 **A 股专属事件词典 + 权威信源加权**；国际快讯被 ×0.3 折扣且仅涉华才计，不再把"海外加息/冲突"当成 A 股利空 |
| "一直显示冻结开仓" | 只有真正的 **S0/S1 重大利空**（如印花税真的上调、千股跌停级事件）才会把市场分打到 -2；普通国际新闻不再误冻，且盘中文案改为"仅供参考、不断言冻结"（见 4.2-P0-4） |
