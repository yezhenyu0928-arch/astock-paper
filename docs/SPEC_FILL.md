# SPEC 附录:真实成交建模 + 小资金适配(Fill Model & Capital Fit)v7

> 本附录**升级并覆盖** SPEC.md 模块2撮合算法中的滑点部分,以及各策略的持仓数规则。
> 目的:让模拟净值曲线"故意难看一点但真实",宁可低估收益,不可低估成本。
> 原则:一切成交建模**取保守侧的确定性结果**(不掷骰子美化),随机模拟仅用于验证阶段的敏感性分析。

## F1 成交概率与冲击成本模型(替换原固定滑点)

只有日线数据时,对"次日开盘价成交"做如下建模:

### F1.1 可参与成交额估计
```
amount_open = 当日 amount × open_frac        # 开盘时段占全日成交比例
open_frac:stock=0.08, etf=0.10               # 保守取开盘半小时下限,config可调
participation p = 订单金额 / amount_open
```

### F1.2 分段滑点(单边、保守方向:买加卖减)
```
p <= 0.005          slip = base                       # base: etf 0.0005, stock 0.0015
0.005 < p <= 0.02   slip = base + impact_k × sqrt(p/0.01)
                    # impact_k: stock 0.0020, etf 0.0008
                    # 即个股 p=1% 时额外 20bp,p=2% 时额外 28bp
p > 0.02            当日只成交到 p=0.02 对应的金额上限(部分成交),
                    剩余部分 status='cut_liquidity' 直接放弃并写日志,
                    reason 追加 '[流动性截断,原计划X元仅成交Y元]'
                    ⚠ 不顺延剩余——手动跟单者不会连续多日追同一笔,模拟应一致
```
部分成交同样推送给用户,让你实盘也只买截断后的量。

### F1.3 极端行情成交概率(确定性保守规则)
| 情形(以次日开盘判断) | 买单 | 卖单 |
|---|---|---|
| 一字涨停:open ≥ limit_up×0.998 | cancelled(成交概率≈0) | 正常成交(涨停卖出无碍) |
| 一字跌停:open ≤ limit_down×1.002 | 正常成交 | deferred 顺延,连续5日仍跌停→按第6日开盘强制成交并如实记录深亏 |
| 高开>3%(相对昨收)的买单 | 成交价额外 +0.2% 追价成本 | — |
| 低开>3% 的卖单 | — | 成交价额外 −0.2% |
| 停牌 | deferred(沿用原规则) | deferred |
⚠ 原 SPEC"昨收涨停则买单作废"规则保留,两道判断都要过。

### F1.4 验证阶段的蒙特卡洛(validate.py 增补)
五关验证第④关"成本加压"升级为:impact_k×2 + open_frac×0.5 双重加压;另增第⑥关:对成交价在 [开盘价, 开盘价×(1+2×slip)] 均匀抽样,跑 200 次蒙特卡洛,报告净值终值的 5% 分位——**以 5% 分位而非均值作为策略入池依据**。

## F2 小资金适配(1–10 万为默认设计目标)

### F2.1 硬约束(engine 强制执行)
```
min_order_amount = 5000 元(config可调)
  # 最低佣金5元在5000元上=万10,再小的单成本失控;低于此的 buy 直接跳过并记日志
单手门槛剔除:某股 100股×现价 > 总资产×max_position_pct → 该股不可买
  # 例:5万资金×15%上限=7500元,单手价超7500的高价股自动出池(如千元股)
现金缓冲:所有买单合计目标 ≤ 净值×0.98(原S2规则推广为全局)
```

### F2.2 持仓数随资金自适应(替换 registry 固定 hold_n)
```
effective_hold_n = min( registry.hold_n,  floor(capital × 0.98 / min_ticket) )
min_ticket = 8000 元(config可调,单票最低配置额)
```
| 资金 | S1红利(≤10) | S3趋势(≤10) | S4小市值(≤20) | S2/S5(ETF) |
|---|---|---|---|---|
| 3万 | 3 | 3 | 3 | 不受影响 |
| 5万 | 6 | 6 | 6 | 不受影响 |
| 10万 | 10 | 10 | 12 | 不受影响 |
个股权重同步改为 `0.98/effective_hold_n`。⚠ 持仓数减少会放大个股风险与净值波动,验证报告必须按**用户 config 的真实 capital** 跑,不许用100万跑出漂亮曲线给5万的用户看。

### F2.3 资金分档建议(README 用户手册同步此表)
| 你的资金 | 建议开启 | 原因 |
|---|---|---|
| <3万 | 仅 S2 + S5(纯ETF) | 个股策略持仓数太少,单票风险过大;ETF无此问题且滑点最小 |
| 3–10万 | S2/S5 + 至多1个个股策略(推荐S1或S3) | S4小市值在此档流动性冲击相对最大,谨慎 |
| >10万 | 全部可选 | 仍受 F1 流动性截断保护 |
赛马结论只在**同资金档内**可比;换资金额=换 config 重跑,历史曲线归档不混接。

## F3 用户自定义(config.example.yaml 新增 custom 段)

```yaml
custom:
  min_order_amount: 5000        # 单笔最低金额
  min_ticket: 8000              # 单票最低配置额(决定持仓数自适应)
  open_frac: {stock: 0.08, etf: 0.10}
  impact_k: {stock: 0.0020, etf: 0.0008}
  hold_n_override: {}           # 如 {s1_dividend@v1: 5} 手动锁定持仓数(仍受自适应上限约束)
  universe_exclude: []          # 黑名单:你不想碰的代码,如 [sz300xxx]
  s2_universe_extra: []         # 给S2的ETF池追加标的(如你偏好的行业ETF),追加即视为
                                # 注册 s2_etf@v2 新赛道,旧曲线归档——防止悄悄改池美化历史
  push_times_intraday: ["10:00","13:30","14:45"]
  risk_override: {}             # 如 {strategy_max_drawdown: 0.12} 收紧熔断线(只许更严不许放松)
```
⚠ risk_override 实现时校验:任何放松风控阈值的取值(如熔断线>0.15)一律拒绝加载并告警。
