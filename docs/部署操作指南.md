# GitHub Actions 部署操作指南（手把手）

> 目标：重建数据库缓存 → 让线上看板显示最新 7 策略。全程在浏览器点按钮，约 30-60 分钟（多为等待）。

## 前提检查（首次部署或换机才需要）

1. **仓库 Secrets 已配**（推送/告警用）：仓库页 → `Settings` → 左侧 `Secrets and variables` → `Actions` → `New repository secret`，确认已有：
   - `PUSHPLUS_TOKEN`（必填，微信推送）
   - `SMTP_AUTH_CODE`（必填，备用邮件）
   - `ANTHROPIC_API_KEY`（选填，仅开大模型消息档时）
2. **GitHub Pages 已开**：`Settings` → `Pages` → Source 选 `Deploy from a branch` → 分支 `main`、目录 `/docs` → `Save`。

---

## 第 1 步：手动跑 backfill（重建历史数据库）

1. 打开仓库页，点顶部 **`Actions`** 标签。
2. 左侧工作流列表点 **`backfill`**。
3. 右侧出现提示条 “This workflow has a workflow_dispatch event trigger.”，点右边的 **`Run workflow`** 按钮 → 弹出小框，分支保持 `main` → 再点绿色 **`Run workflow`**。
4. 页面出现一条黄色（运行中）记录，点进去可看实时日志。**约 15-40 分钟**（要抓沪深300+中证500+行业ETF+年度ROE）。跑完变绿勾 ✅。
   - 中途某几只股票取数失败会自动重试/跳过，不影响整体，只要最后是绿勾即可。

## 第 2 步：手动跑 daily（生成最新看板 + 推送）

1. 还在 `Actions` 标签，左侧点 **`daily`** → 右侧 **`Run workflow`** → `Run workflow`。
2. 约 3-8 分钟跑完。**手机微信收到“心跳”消息 = 成功**。
3. daily 跑完会自动把新的 `docs/index.html` 提交回仓库并触发 Pages 重新发布。

## 第 3 步：看线上看板

- 打开 `https://yezhenyu0928-arch.github.io/astock-paper/`
- **如果还显示旧的 5 策略：是浏览器/CDN 缓存，不是没更新**。解决：
  - 电脑：按 `Ctrl + F5`（强制刷新）；或无痕窗口打开。
  - 手机：下拉刷新，或换浏览器/清缓存。
  - Pages 发布本身也有 1-3 分钟延迟，稍等再刷。
- 确认看到 **7 个策略卡**（ETF动量轮动 / 红利低波 / 红利低波·质量增强 / 双均线趋势 / 沪深300价值精选 / 大盘估值网格 / 行业ETF轮动）即成功。

## 之后就全自动了

- 每个交易日 **17:40 / 18:10** daily 自动跑（更新数据+算信号+推送+刷看板）。
- 每 3 天 `keepalive` 自动续命数据库缓存；每月 1 日 `probe` 自动体检数据接口。
- 你只需每天看微信推送 + 刷看板跟单。**没收到心跳 = 当天别跟单，去 Actions 看哪步红了**。

## 常见问题
- **backfill 红了**：多为某数据源临时抽风，直接再点一次 `Run workflow` 重跑（幂等，会跳过已完成的）。
- **daily 红了但 backfill 是绿的**：点进 daily 日志看报错；若是“数据库疑似丢失”告警，说明缓存被驱逐，重跑一次 backfill 即可。
- **没收到微信**：检查 `PUSHPLUS_TOKEN` 是否填对、pushplus 是否关注了公众号。
