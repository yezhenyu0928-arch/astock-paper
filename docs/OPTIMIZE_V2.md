## 二、公开研究依据（为什么这样改策略）

1. **红利低波是 A 股过去十年最扎实的低风险因子组合**：中证红利低波全收益指数 2015→2025 累计约 +162%（同期主要宽基第一），近6年年化 13.4% vs 沪深300 的 7.24%；2022-2024 调整期逆势 +42.6%。→ S1 保留并强化，实盘配置的核心。
2. **行业动量轮动有效且是"政策beta"最可靠的免费载体**：西部证券《行业动量策略的CTA思维》2015.01-2024.10 年化 19.51%、最大回撤 17.13%、夏普 1.06；政策利好（如设备更新、化债、AI扶持）最终都会体现为行业指数动量，行业ETF轮动可自动捕捉，无需爬政策文本。→ 新增 S6（卡C）。
3. **A 股长期有效因子**（多家公开研究一致）：小市值、红利、低波动、质量（盈利+低杠杆）长期有效；短期反转长期强于动量，但 2017 年后机构化使动量渐强、反转渐弱；高换手个股未来收益差。→ S1 v2 加质量因子（卡D）；不引入短反转（换手/摩擦对5万资金不友好）。
4. **北向资金实时数据 2024-08 起停止披露**，政策文本量化无免费稳定源。→ "政策/行业面"以行业ETF动量为主承载 + 已有 news 层做事件降险，不做爬政策新闻选股（不可靠且难验证）。

### 卡A（P0·截止7-05）：看板 V2 —— report_html.py 全面改版

只改 `report_html.py`（模板与生成逻辑），不动引擎/策略。产出仍是零外部依赖、离线可开、手机自适应的静态页。

**A1. 全局色规范（盈红亏绿，A股习惯）**
- CSS 变量：`--up:#d92b2b`（涨/盈利/买入）、`--down:#0a9e6b`（跌/亏损/卖出）、中性灰不变。
- 所有收益率数字带符号（+1.2% / -0.8%），≥0 用 --up，<0 用 --down；熔断徽章保持红底白字但文案"熔断"。
- 今日操作卡：买入=浅红底深红字，卖出=浅绿底深绿字（与现状恰好对调）。
- trades.html 同步：买入行红、卖出行绿。
- 图表：策略曲线终值≥0 红线、<0 绿线；基准线灰虚线。

**A2. 页面结构（自上而下）**
1. 标题 + 副标行（生成时间 · 数据最新交易日）+ **数据新鲜度横幅**：
   用库内交易日历计算 `max(daily_bar.trade_date)` 距今落后的**交易日数**（非自然日）：
   落后≤1 不显示；=2 黄色横幅"⚠ 数据延迟1个交易日"；≥3 红色横幅"🛑 数据已过期，今日暂停跟单，等待系统恢复"。
2. **【今日操作】聚合区（置顶）**：跨策略汇总所有 pending 订单，列：策略/方向/代码/名称/数量描述/参考价/理由。
   右上"📋 复制指令"按钮：原生 JS `navigator.clipboard.writeText`，格式每行一条
   `买入 510500 中证500ETF 约4900股 参考价7.85（S2 ETF动量轮动）`，无操作则显示"今日无操作"。
   **实时价渐进增强（用户需求：按当天实时价判断可行性）**：页面加载时对操作区标的动态插入
   `<script src="https://qt.gtimg.cn/q=sh510500,...">`（腾讯行情，返回 JS 变量赋值、天然跨域可用、GBK 用 text 解析或仅取数字字段），
   成功则把参考价替换为"实时价 X（HH:MM）"并按实时价重算股数与预估金额，涨跌幅红绿标色；
   **失败/超时(3s)静默降级**回昨收参考价并显示"（昨收参考）"。此增强失败不得影响页面任何其他功能（try/catch 包裹，无 Promise 悬挂）。
   每条操作附**跟单价格带**：买入=参考价×[0.99, 1.02] 区间内可跟、高于上带减半或放弃；卖出=不设下带、竞价直接卖。带宽写成常量便于调整，规则文案渲染在操作卡下方一行小字。
3. **【实盘赛马总览】表（2026-07-06 起算）**：列=策略 | 实盘累计收益率(红绿) | 实盘最大回撤 | 最新净值 | 持仓数 | 状态(🟢/🔴熔断) | 回测参考(2022→今累计·Calmar·判定)。
   实盘列数据来自 nav_history（见 A4 锚定规则）；回测参考列沿用现有 _backtest_summary。
4. **【策略卡】×N**，每张含：

   b. 一句话定位 + **因子与权重表**（内容见 A5 字典，含调仓频率/持仓数/适合资金档）；
   c. `<details>`（默认收起）summary=`📈 实盘收益率曲线（07-06起）当前 +x.x%`，展开为 **SVG 大图**（见 A4）；

2. **cache 防驱逐保活**：新增 `.github/workflows/keepalive.yml`：cron 每3天一次（如 `17 3 */3 * *`），步骤=restore DB cache（restore-keys: db-）→ 校验文件存在 → 以新 key `db-keepalive-${{ github.run_id }}` save。GitHub cache 7天未访问驱逐 + 10GB 限额，此举覆盖春节/国庆长假。同时在 README 部署自检节补一行说明。
3. **push 失败不再吞错**：daily.yml `commit snapshot` 步改为：push 失败重试（pull --rebase 后再 push，最多3次），仍失败 → `python notify.py --alert "🔴 看板快照推送失败，Pages 未更新"`（告警失败本身 `|| true`）。
4. **看板生成失败要告警**：run_daily.py 中 `report_html.generate()` 的 except 分支补一条 notify 告警（不阻断主流程返回码）。

### 卡H（P1·第一周内）：逐股买卖理由增强（每笔操作"知其所以然"）

**硬约束（本卡红线）**：只允许改各策略文件中 reason 字符串的构造（及为此需要的局部变量），**禁止改变任何过滤条件、排序、权重、下单逻辑**。理由文案变化不构成 registry"改参数需注册新版本"——选股行为必须零变化，验收须证明。

1. reason 模板统一为「动作依据 + 关键数字 + 池内排名」：
   - S1 买入 `红利低波:买入XX(股息率5.2%第3/28·波动率池内最低12%)`；卖出按实际原因分支：`调仓:XX综合排名第15掉出前6,卖出` 或 `不再满足门槛(股息率3.1%<4%),卖出XX`。
   - S3 买入 `MA20上穿MA60,量比2.3倍,买入XX(站上MA60幅度+4.1%)`；卖出 `收盘9.85跌破MA20(10.02),清仓XX`。
   - S4 买入 `小市值价值:买入XX(市值212亿第5小·PB0.9·20日+2.1%·综合分第2/400)`；卖出补排名，同 S1。
   - S2 买入补双窗口数值 `(r20=+5.3% r60=+11.2%,6只中第1)`；S5 已含档位与PE分位，不动；S6（卡C后）按 S2 同格式。
2. 展示端配合：微信推送与看板"今日操作"完整显示 reason 不截断；trades.html 实盘区去掉 [:44] 截断（与卡A A3 呼应，若卡A已做则跳过）。
3. 持仓"为什么还拿着"：卡A 持仓表"买入理由"列自动获益于本卡更详尽的 reason，无额外工作。
